Overview

EDU Results Publishing is built for education websites that need a WordPress-native way to manage students, subjects, exams, boards, session years, departments/groups, and searchable result sheets.

  • Admins enter student and result data from the WordPress dashboard.
  • Students search from the frontend using academic identifiers.
  • Result sheets display institution branding, student details, subject marks, grades, GPA, and print controls.

Setup Checklist

  1. Install and activate the plugin from WordPress.org or from the GitHub release/source package.
  2. Open plugin settings and add institution logo, institution name, registration number, contact details, website, and result page heading.
  3. Create session years, examinations, boards, and departments/groups.
  4. Add subjects and assign subject codes.
  5. Add student records with ID number, registration number, personal details, and taxonomy assignments.
  6. Create result records, select the student registration, add subject marks/grades, GPA, and result status.
  7. Publish a frontend search page using the result search shortcode.

Frontend Shortcode

Add the result search interface to any WordPress page or post using the plugin shortcode. Confirm the exact shortcode from the plugin readme/source before publishing final public documentation.

[edu_result_search]

The frontend form supports examination, year/session, board, department/group, registration number, and roll number fields.

Admin Workflow

  • Students: Store ID, registration number, DOB, gender, blood group, contact details, guardian/family fields, and document numbers.
  • Subjects: Create a subject title and subject code for result sheet rows.
  • Results: Select student registration, add roll number, result status, GPA, GPA without additional subject, student type, and subject marks/grades.
  • Settings: Configure institution identity and result sheet branding.

Data Structure

The plugin is organized around WordPress admin content and taxonomies so data can be managed with familiar dashboard screens.

  • Students hold student profile and identification fields.
  • Subjects hold subject names and subject codes.
  • Results connect students to marks, grades, GPA, and status.
  • Session Years, Examinations, Boards, and Departments/Groups organize result filtering.

WordPress setup

  1. Download or install the plugin from WordPress.org.
  2. Activate it from Plugins → Installed Plugins.
  3. Configure institution settings and academic taxonomies.
  4. Add students, subjects, and result records.
  5. Create a public page and add the result search shortcode.
